# Daisy Delay VST3/Standalone (macOS test harness)

This is a JUCE wrapper around the existing firmware DSP/mode code so you can audition the effect before flashing hardware.

## What is reused from firmware
- Delay modes and processing implementation in `src/modes/*`
- DSP building blocks in `src/dsp/*`
- Parameter mapping in `src/params/*`

## Build (macOS)

```bash
cd desktop/vst
cmake -B build -S . -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
```

Artifacts are generated by JUCE in the build tree (VST3 + standalone app).

## Install targets (macOS)

Install VST3 to user plugin folder:
```bash
cmake --build build --target install-user-vst3
```

Install standalone app to `~/Applications`:
```bash
cmake --build build --target install-user-standalone
```

Install both:
```bash
cmake --build build --target install-user
```

## UI Controls
- 7 rotary controls: `Time`, `Repeats`, `Mix`, `Filter`, `Grit`, `Mod Spd`, `Mod Dep`
- Mode selector dropdown for all 10 delay modes
- Bypass toggle

## Notes
- For best behavioral match with firmware, run your host at **48 kHz**.
- Daisy-only hardware layers (GPIO, OLED, relay hardware, MIDI learn UI, etc.) are not part of this harness.
